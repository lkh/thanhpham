Portfolio Filter Help

About Portfolio Filter
A Portfolio is a top level grouping of cases within the APM system. The Portfolio Filter is a collection of rules that are applied to new cases inserted into APM. Cases do not need to be assigned to a Portfolio, however we recommend always using a default Portfolio. There are reporting and maintenance advantages of using Portfolios to manage cases.
About Rules
  • A Rule is defined as an IF <condition(s)> THEN <action(s)> statement.
  • A Condition is a statement that can be tested to see whether it is true or false. A Condition is specified as: <Value> <Operator> <Value>.
  • A value in a Condition can either be a variable, a static string or a function. All user-typed values are static strings and highlighted in NORMAL font. All variables are highlighted in BOLD font. Variables can be dragged and dropped into Condition fields from the right-hand panel of the rule editor, or can be typed in using the auto-complete feature.
  • An Action is what we want to perform if the evaluated condition is true. To define more than one action for a rule, click on the icon.
  • The following types of values can be used in conditions:
    • Input Variables - These are fields from the input payload that have been set up for use in rules.
    • Input Functions - These are values calculated from the fields in the input payload.
    • General Functions - These are functions offered by APM that allow advanced rules to be created. Function arguments can be static text (e.g. '100' or 'EC1') or can be direct references to fields in a payload (e.g. input.collection.amountDue).
    • Result Codes - These are individual or groups of result codes.
    • Historic Results - These are attributes that consider the results across all alerts and include the best result.
    • Current Results - These are attributes similar to historic results, but are since the last action time.
    • Fixed Values
  • The following operators are available:
    • Equals - Tests for equality.
    • Not Equals - Tests for inequality.
    • Greater Than - Performs a comparison of two numeric values.
    • Less Than - Performs a comparison of two numeric values.
    • In - True if a given value is IN a group of values.
    • Not In - True if a given value is NOT IN a group of values.
  • The basic actions are:
    • Terminate Case
    • Suspend Case [Duration]
    • Goto Portfolio <Portfolio>
    • Goto Strategy <Strategy> [Delay]
    • Set <Setting> <Value>
    • Create Alert [Delay]
    • Set Delay [Delay]
    • Clear Settings
About Actions
  • Terminate Case - stops processing a case, removes all input values and passes a terminate event to the AdeptraLink.
  • Suspend Case [Duration] - stops processing a case while APM keeps a record of its current input values. A suspend action lasts until either the duration specified in the Suspend Case action (if supplied) is reached, or the case re-enters the system. APM will Terminate the case if a Suspend Case action expires.
  • Goto Portfolio <Portfolio> - assigns the case to the specified Portfolio.
  • Goto Strategy <Strategy> [Delay] - applies the selected Strategy to subsequent alerts for that case. If a delay is specified, these alerts will be submitted after the delay period has expired. Note that the case will not initiate a refresh before a delayed Goto Strategy is executed; also a refresh push will not effect a Goto Strategy action that has already been scheduled.
  • Set <Setting> <Value> - sets a value for an individual setting. Settings specified using Set actions can be combined with predefined strategies (known as "user" strategies). Settings specified using SET actions take priority over predefined strategy settings. Where Set actions are included, the combination of settings applied as a result is known as an "auto strategy".
  • Create Alert [Delay] - stops filter execution and creates an alert with the current settings. If a delay is specified, the alert will be submitted after the delay period has expired. If no Set actions have taken place before a Create Alert action, then the application default settings will be used.
  • Set Delay [Duration] - provides the ability to set a delay prior to associating the case with a strategy or an alert. This may be useful when it is desired to specify different delay values based on matching rule conditions. A delay specified in a Set Delay action will override any delay value specified in any subsequent Goto Strategy and Create Alert actions.
  • Clear Settings - clears all settings set so far.
  • Wait For Result - stops processing a case until results for the case are received by APM. The case will remain active with the input values kept. When a Case Result is received, the rules will be rerun from the main portfolio with a Current Case Execution Type of Received Case Result.
Add a Rule
If there are no existing rules in the Portfolio Filter, the "Add a Rule" form will automatically be displayed. Once Rules have been created, clicking on the icon in an existing rule will open the form below the current rule. The new rule will be inserted at that point in the filter priorities. For help with constructing rules, please click on the contextual Help link within the form.
Copy a Rule
To copy an existing rule, click the icon. The new rule will be inserted below the rule that has been copied, with the Rule editing form open. For help with constructing rules, please click on the contextual Help link within the form.
Edit a Rule
To edit an existing rule, click the icon to open the Rule editing form. Operations on other Rules are disabled while the edit form is displayed. For help with constructing rules, please click on the contextual Help link within the form.
Delete a Rule
To permanently delete a rule, click on the icon. You will be asked for confirmation before the Rule is deleted from the Portfolio Filter.
Enable or Disable a rule
The checkbox in the "Enable" column of each Rule can be used to enable or disable the rule. Disabled Rules are displayed grayed out and will not be evaluated when the filter is applied to new cases.
Setting Rule Priorities
The Rules in the Filter are applied in the listed order. Once a match is found for a case, that Rule's action is applied, and further rules are NOT processed. The order of the rules can be changed using the Up and Down arrows in the Priority column of each Rule, or by clicking in the Rule's Priority column and dragging the Rule into the desired position in the table.

Filter Help

About the Rule Editor
The Rule Editor allows the Rules in a Filter to be named and defined. Once you are happy with the rule you have defined, click "Save" to save your changes. Clicking the "Cancel" button at any time will discard any changes you have made and close the form.
Define a Condition
  • The value in a Condition can be a variable, a function or a static string.
    • Variables: The right-hand panel of the Rule Editor contains a list of values that can be used in Rule Conditions. To use one of these in a Condition, click on the required item and drag it across to the required field in the Condition definition. Alternatively, you can type in the value directly. An auto-complete feature will help by displaying a drop-down list of values it recognizes. When variables are used as values, they are highlighted in BOLD font. The draggable list items should be used where these are available, to ensure that the precise naming and spelling is used.
    • Functions: The right-hand panel of the Rule Editor contains a list of values that can be used in Rule Conditions. To use one of these in a Condition, click on the required item and drag it across to the required field in the Condition definition. Alternatively, you can type in the value directly. An auto-complete feature will help by displaying a drop-down list of values it recognizes. When variables are used as values, they are highlighted in BOLD font. The draggable list items should be used where these are available, to ensure that the precise naming and spelling is used.
    • Static text: Static text values can also be typed directly into the Condition fields, e.g. where a numeric value is required. These values are considered to be static strings and are highlighted in NORMAL font.
  • The operator to be used to evaluate the Condition can be selected from the drop-down list in the center of the Condition. The following operators are available:
    • Equals - Tests for equality.
    • Not Equals - Tests for inequality.
    • Greater Than - Performs a comparison of two numeric values.
    • Less Than - Performs a comparison of two numeric values.
    • In - True if a given value is IN a group of values.
    • Not In - True if a given value is NOT IN a group of values.
Add a Condition
Further conditions can be added to the rule by clicking on the icon to the right of an existing Condition. Multiple conditions within the same rule can joined together using the available logical operators. A logical operator dropdown with OR and AND options is available for selection located at the left of each Condition. At runtime, OR operators get precedence over the AND i.e sequence of ORs are evaluated before the ANDs. In display mode, Condition column shows sequence of ORs in a light yellow background to highlight the evaluation order. e.g : C1 AND C2 OR C3 OR C4 AND C5 would be evaluated as C1 AND ( C2 OR C3 OR C4 ) AND C5
Delete a Condition
A Condition can be deleted by clicking on clicking on the icon to the right of the Condition. A Rule with no conditions will always be executed.
Define an Action
The Action defines what should happen if the Rule is matched, i.e. all of its Conditions evaluate to true. The following Actions are available:
  • Terminate Case - stops processing a case, removes all input values and passes a terminate event to the AdeptraLink.
  • Suspend Case [Duration] - stops processing a case while APM keeps a record of its current input values. A suspend action lasts until either the duration specified in the Suspend Case action (if supplied) is reached, or the case re-enters the system. APM will Terminate the case if a Suspend Case action expires.
  • Goto Portfolio <Portfolio> - assigns the case to the specified Portfolio.
  • Goto Strategy <Strategy> [Delay] - applies the selected Strategy to subsequent alerts for that case. If a delay is specified, these alerts will be submitted after the delay period has expired.
  • Set <Setting> <Value> - sets a value for an individual setting. Settings specified using Set actions can be combined with predefined strategies (known as "user" strategies). Settings specified using SET actions take priority over predefined strategy settings. Where Set actions are included, the combination of settings applied as a result is known as an "auto strategy".
  • Create Alert [Delay] - stops filter execution and creates an alert with the current settings. If a delay is specified, the alert will be submitted after the delay period has expired. If no Set actions have taken place before a Create Alert action, then the application default settings will be used.
  • Set Delay [Duration] - provides the ability to set a delay prior to associating the case with a strategy or an alert. This may be useful when it is desired to specify different delay values based on matching rule conditions. A delay specified in a Set Delay action will override any delay value specified in any subsequent Goto Strategy and Create Alert actions.
  • Clear Settings - clears all settings set so far.
  • Wait For Result - stops processing a case until results for the case are received by APM. The case will remain active with the input values kept. When a Case Result is received, the rules will be rerun from the main portfolio with a Current Case Execution Type of Received Case Result.

After selecting the action type from the drop-down any optional or required information to complete that action definition should be entered using a combination of the drop-downs (to select Portfolios, Strategies, or Settings) or text entry boxes (to enter delays, durations, or other values).

Where required, delay periods should be specified as a number of days, hours, and minutes, in the format "0d0h0m", where each component is optional. Example delays could be: "1d", "1d12h", "1h30m", "48h". It is also possible to choose a time of day for the action to delay until. If required, this should be specified after the delay length, in 24 hour format. For example:

  • "24h 15:00" would delay for 24 hours and then again until the next time it is 3:00pm.
  • "0m 08:00" would delay until the next time it is 8:00am.
The specified time will be interpreted relative to the timezone configured by the administrator.

Add an Action
Further actions can be added to the rule by clicking on the icon to the right of an existing Action. This is useful when multiple Strategy settings need to be set in a single rule by using multiple Set actions.In a multi action scenario, the actions are executed in the given order until a final action is encountered. A final action is one of: Terminate Case, Suspend Case, Goto Portfolio, Goto Strategy or Create Alert. Any actions that exist after a final action is executed will be ignored.
Delete an Action
If the Rule has two or more Actions, an Action can be deleted by clicking on clicking on the icon to the right of the Action. A Rule must always have at least one Action, so if there is only one remaining Action, it can be edited but not deleted.
